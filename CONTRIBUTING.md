# Contributing to This Library

Thank you for wanting to contribute. This library is specifically built 
for non-technical users — so contributions from non-developers are 
especially welcome.

You do not need to know how to code to contribute here.

---

## What we are looking for

**New prompts** that solve real problems for researchers, writers, 
educators, or professionals — tested and explained in plain language.

**Improvements to existing prompts** — better wording, clearer 
instructions, additional tips based on real use.

**New guides** on using Claude effectively for specific use cases 
or audiences.

**Corrections** — if something is wrong, unclear, or outdated, 
please flag it.

---

## How to contribute a prompt

Every prompt in this library follows the same format. Please use it:
```
## Prompt Name

**What it does:** One sentence description.

**Best for:** Who benefits most from this prompt.

**When to use:** The specific situation where this prompt works best.

**Copy this prompt:**
[THE ACTUAL PROMPT WITH [BRACKETS] FOR VARIABLE PARTS]

**Tips:**
- Practical advice for getting better results
- What to do after the initial response
- Common mistakes to avoid
```

---

## How to submit

1. Open an issue describing the prompt or improvement you want to add
2. If you are comfortable editing files directly on GitHub, you can 
   submit a pull request
3. If GitHub feels too technical, paste your contribution in an issue 
   and I will add it

No contribution is too small. A single well-tested prompt that solves 
a real problem is genuinely valuable.

---

## Standards

Every prompt must be:
- Tested — you have actually used it and it works
- Explained — what it does and when to use it, in plain language
- Formatted — following the template above
- Honest — if a prompt has limitations, say so in the tips

---

## Questions

Open an issue with your question. Response time is typically within 
a few days.
